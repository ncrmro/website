Recently a client i've been working with has a binary that process work that can consume all the CPU it can throw at it, between 3-7 cores. 

This task is not easily parallelized and thus we need do vertical scaling up to the max of our currently available node size. If this process can consume more we scale up a new node from 0 to 1 and keep bring it up with bigger CPU/RAM until we find the limitation.
